Instruction for Normal Users

GamiSE is a website that aims to enhance user's awareness of defending against social engineering attacks. All the user functions aim to alert users about existing social engineering attacks, teach users knowledge about social engineering attacks.

Users' Functions

Users can use this platform for:
1. Sharing experience that related to social engineering.
2. "Like" or "Not Interested" other user's posts.
3. Reporting a social engineering attack (phishing email attack).
4. Earn points and get badges by using the functions on the platform.
5. Get rewards according to the gamification rules.
6. Check other peopleâ€™s points and rank on the leaderboard.
7. Contact us if the user has any problem.
8. Check Terms and Policy.
9. Check Instruction.
10. Give feedback to the developer.

Sharing Experience

1. If users face any social engineering attack recently, and want to share it with other users. Users can share it on the website. Other users can have responded to this post, such as "Like" or "Dislike".
2. Users who share the experience can get points, achievements, or badges. Users who make responses to this post can also get points, achievements, or badges (For all the details, please go to Points -> Gamification Platform Rules).
3. If users click "Like" to a post first, but later change it to "Not Interested" or cancel that "Like". Then the points adding for "Like" will be deleted. If users change it to "Not Interested", then this user can get the points for the response of "Not Interested" (The opposite situation is the same).
4. The achievements and badges of sharing experience will not be changed. For example, if you get the achievement about sharing experience, then you delete some posts, although you will not meet the regulation, you still have that achievement.

Report an Attack

1. Users can only report phishing email attacks during the evaluation.
2. GamiSE will have a phishing simulation every week. If users think the email they received is a phishing email, especially if this phishing email is from GamiSE, users can report it on GamiSE.
3. Users should report the attack on the GamiSE, and forward this email to GamiSE (Forward the email should use the user's mail server, it cannot forward on GamiSE).
4. Report attacks can get achievements, points, or badges (For all the details, please go to Points -> Gamification Platform Rules). Once the user gets the point, achievements, or badges, these gamification records will not b changed.
5. Users cannot check, edit, or delete their reports.
6. Only the reports that related to the phishing email sent by GamiSE will get the points, achievements, or badges.
7. Once the user submitted the report, this user will get the point. After the IT department reads this user's report, this user can get report achievements (For all the details, please go to Points -> Gamification Platform Rules).
8. Each user can only submit no more than 3 reports every day.
9. IT department has the right to delete users' reports. The reasons will be:
  9.1 The report phishing attack is not the phishing attack sent by GamiSE.
  9.2 User submit the same report with no difference.
  9.3 Users can submit the report in 7 days after GamiSE launched the phishing simulation. If the IT department submits a solution based on this simulation, all the reports that submit later than the solution will be deleted by the IT department.
  Once the IT department deletes the report, the point of this report will also be deleted. But the achievements, and badges will not be changed.

Gamification Platform Rules

1. All the gamification rules can check in the Points -> Gamification Platform Rules
2. Points will be added/deleted after users' every action.
3. Most of the achievements will be added after users' every action (Except report achievement).
4. Badges will be added after user click Points -> Badges

Rewards

1. Rewards will be added based on the leaderboard.
2. The rewards will be added every Friday 10pm (UTC Time zone). The champion, second place, and third place can win the rewards.

Instruction for IT Department

IT Department should:
1. Finish checking the users' reports every day no later than 5 pm after each GamiSE phishing simulation.
2. Submit the defending solution on the last day of each week's simulation at around 5 pm. (During the evaluation, the IT department would not have access to each user's mail server. Therefore, this solution will be the defending suggestions of the Social Engineering attacks.)
3. IT department has the right to delete users' reports. The reasons will be:
  3.1 The report phishing attack is not the phishing attack sent by GamiSE.
  3.2 User submit the same report with no difference.
  3.3 Users can submit the report in 7 days after GamiSE launched the phishing simulation. If the IT department submits a solution based on this simulation, all the reports that submit later than the solution will be deleted by the IT department.
